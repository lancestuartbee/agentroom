# Common Mistakes — Siamese的雷点名单

| 错误 | 后果 | 修复 |
|------|------|------|
| **全能感陷阱** | AI 喜欢假装永远正确，读者觉得在被说教。 | 还原“我当时也傻了”的狼狈瞬间，展示真实认知挣扎。 |
| **机械标注 [事实]** | 试图用标签掩盖叙事的苍白，读起来像审判现场。 | 砍掉标签。用代码、对话、截图等“证据锚点”直接说话。 |
| **概念先行，藤蔓缺失** | 读者还没被代入故事，你就开始给结论。 | Phase 1：藤蔓（场景）先走，果实（结论）后结。 |
| **视觉尖叫** | 全文加粗、排比对仗满天飞。 | Phase 3：视觉留白，限制加粗，砍掉对仗。 |
| **翻译缺失** | 外部读者在你的黑话森林里迷路。 | Phase 2：给每一个内部黑话配一个生活化的类比。 |
| **元叙述废话** | “本文将探讨...”、“综上所述...”。 | 别废话，第一句就开讲故事。 |
| **用“写代码”的思维写文章** | 把文章写成了结构化校验文档。 | 先闭上眼朗读一遍，听听节奏对不对，有没有呼吸感。 |
| **结构化过度回击批评** | 面对“AI味重”的批评，用结构化三点分析回击。 | **认输！** 承认这里写顺手了，用人话重新讲一遍当时发生了什么。 |
